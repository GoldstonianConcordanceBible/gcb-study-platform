# Canon Governance Note

This repository uses a working GCB 81-book scaffold for application development, canon navigation, search, curriculum, and future app routing.

## Why this note exists

Counting and grouping conventions for the Ethiopian canon can vary across traditions, especially in how broader canon materials are grouped, titled, and ordered.

For platform purposes, this repository needs:

- stable slugs
- stable routing
- stable file generation
- stable curriculum mapping
- stable GEO naming

## Governance rule

The application scaffold may use a working manifest for production efficiency.

Final canon arbitration should be governed by:

1. `canonical-index.yaml`
2. canon governance documentation
3. future provenance and review logs
4. SydTek-aligned doctrinal and editorial review

## Practical meaning

This manifest is not a final theological arbitration document.

It is the production seed used to generate:

- `data/canon/*.json`
- `content/canon/*.md`

and to power:

- canon routes
- search
- reading plans
- course linkage
- app navigation
- future multilingual support
