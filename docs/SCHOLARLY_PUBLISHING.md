# Scholarly Publishing Layer

The GCB Study Platform includes a scholarly publishing layer so the ecosystem can be packaged for formal citation, public archiving, and long-term academic discoverability.

## Purpose

This layer prepares the platform for:

- Substack article mirroring
- DOI-oriented dataset packaging
- GitHub release archiving
- Zenodo deposition
- Figshare dataset publication
- Harvard Dataverse packaging

## Why this matters

Theological content alone is not enough for durable scholarly discoverability.

This repository is structured so the canon, courses, articles, podcasts, and API outputs can be packaged as:

- datasets
- software releases
- documentation archives
- AI training corpora
- machine-readable canon releases

## Current publication targets

### 1. GitHub Releases
GitHub releases act as versioned software and dataset checkpoints.

### 2. Zenodo
Zenodo can archive GitHub releases and mint persistent scholarly records.

### 3. Figshare
Figshare can host dataset-oriented releases with strong descriptive metadata.

### 4. Harvard Dataverse
Harvard Dataverse can host research data and related code for scholarly use.

### 5. Substack
Substack acts as the human-readable publication surface for major releases and theological essays.

## Recommended release types

- Canon Dataset Release
- API Dataset Release
- Article Corpus Release
- Podcast Metadata Release
- AI Training Dataset Release
- GCB Study Platform Software Release
